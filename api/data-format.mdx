## Overview

The CO2 API returns a simple, structured JSON response containing real-time CO2 measurements and timestamp information. This page documents the data format, parsing logic, and how data flows through the application.

## Response Structure

### Root Object

The API returns a JSON object with two fields:

<ResponseField name="co2" type="number" required>
  **CO2 Concentration in PPM**
  
  A numeric value representing the carbon dioxide concentration measured in parts per million (ppm). This is the primary measurement value used for monitoring and visualization.
  
  - **Type**: Integer or Float
  - **Unit**: Parts per million (ppm)
  - **Typical Range**: 400-2000 ppm for indoor environments
  - **Usage**: Displayed in real-time dashboard and plotted on time-series chart
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  **Measurement Timestamp**
  
  A formatted string representing when the CO2 reading was taken. Uses a custom date-time format that requires parsing.
  
  - **Format**: `DD/MM/YY, HH:MM A.M./P.M.`
  - **Example**: `"13/03/26, 10:30 A.M."`
  - **Components**: Day/Month/Year, Hour:Minute Period
  - **Clock**: 12-hour format with A.M./P.M. designation
</ResponseField>

### Complete Response Example

```json
{
  "co2": 450,
  "timestamp": "13/03/26, 10:30 A.M."
}
```

## Timestamp Format Specification

### Format Pattern

```
DD/MM/YY, HH:MM A.M./P.M.
```

### Components Breakdown

| Component | Description | Example | Notes |
|-----------|-------------|---------|-------|
| `DD` | Day of month (2 digits) | `13` | Zero-padded |
| `MM` | Month (2 digits) | `03` | Zero-padded, 01-12 |
| `YY` | Year (2 digits) | `26` | Represents 2026 |
| `HH` | Hour (2 digits) | `10` | 12-hour format |
| `MM` | Minute (2 digits) | `30` | Zero-padded |
| `A.M./P.M.` | Period | `A.M.` or `P.M.` | With periods |

## Timestamp Parsing

### The formatTime Function

The application includes a custom `formatTime` function to parse the timestamp format and convert it to a localized time string suitable for chart display:

```javascript
const formatTime = (timestamp) => {
    const [datePart, timePartWithPeriod] = timestamp.split(", ");
    const [day, month, year] = datePart.split("/");
    const [timePart, period] = timePartWithPeriod.split(" ");
    let [hours, minutes] = timePart.split(":");
    
    // Convert 12-hour to 24-hour format
    if (period.toLowerCase() === "p.m." && hours !== "12") {
        hours = parseInt(hours) + 12;
    } else if (period.toLowerCase() === "a.m." && hours === "12") {
        hours = "00";
    }
    
    // Construct ISO date string
    const isoDate = `20${year}-${month}-${day}T${hours}:${minutes}:00`;
    const date = new Date(isoDate);
    
    // Validate date
    if (isNaN(date)) {
        console.error("Fecha no válida:", timestamp);
        return "Invalid Date";
    }
    
    // Format for display
    return date.toLocaleTimeString('es-MX', {
        hour: '2-digit',
        minute: '2-digit',
        second: '2-digit',
        hour12: false
    });
};
```

### Parsing Steps

<Expandable title="Step-by-Step Timestamp Conversion">

1. **Split Date and Time**: Separate on comma-space delimiter
   ```javascript
   "13/03/26, 10:30 A.M." → ["13/03/26", "10:30 A.M."]
   ```

2. **Parse Date Components**: Split date part on forward slashes
   ```javascript
   "13/03/26" → day: "13", month: "03", year: "26"
   ```

3. **Parse Time Components**: Split time with period
   ```javascript
   "10:30 A.M." → timePart: "10:30", period: "A.M."
   ```

4. **Extract Hours and Minutes**
   ```javascript
   "10:30" → hours: "10", minutes: "30"
   ```

5. **Convert to 24-Hour Format**
   - P.M. (not 12): Add 12 to hours
   - A.M. and 12: Set hours to "00"
   - Otherwise: Keep as-is

6. **Build ISO Date String**
   ```javascript
   "2026-03-13T10:30:00"
   ```

7. **Create Date Object and Validate**
   ```javascript
   new Date("2026-03-13T10:30:00")
   ```

8. **Format for Locale**
   ```javascript
   "10:30:00" // 24-hour format for es-MX locale
   ```

</Expandable>

### Time Conversion Logic

<Note>
**12-Hour to 24-Hour Conversion Rules:**

- **1:00 P.M. - 11:59 P.M.**: Add 12 hours (e.g., 2:30 P.M. → 14:30)
- **12:00 P.M. - 12:59 P.M.**: Keep as-is (12:xx remains 12:xx)
- **1:00 A.M. - 11:59 A.M.**: Keep as-is (e.g., 9:15 A.M. → 09:15)
- **12:00 A.M. - 12:59 A.M.**: Convert to 00:xx (midnight hour)
</Note>

## CO2 Value Format

### Data Type and Range

The `co2` field contains a numeric value in parts per million:

```javascript
const { co2, timestamp } = response.data;
setLatestCO2(co2);  // Used directly as a number
```

### Display Format

The CO2 value is displayed with the "ppm" unit suffix:

```javascript
<h1>{latestCO2 ? `${latestCO2} ppm` : "Cargando..."}</h1>
```

**Example output:** `450 ppm`

### Chart Integration

CO2 values are added to a rolling array for visualization:

```javascript
setCo2Data((prevData) => {
    const newData = [...prevData, co2];
    if (newData.length > 10) newData.shift();
    return newData;
});
```

### 1. API Request


### 3. Data Processing

**CO2 Value:**
```javascript

```

**Timestamp:**
```javascript

```

### 4. State Updates

**CO2 Data Array:**
```javascript
setCo2Data((prevData) => {
    const newData = [...prevData, co2];
```

```javascript
setTimestamps((prevTimestamps) => {
    const newTimestamps = [...prevTimestamps, formatTime(timestamp)];

});
```

## Data Validation

### Response Validation

Always destructure and validate the response:

```javascript
const response = await axios.get("https://co2-api.vercel.app/api/co2");
const { co2, timestamp } = response.data;

// Both fields should exist
if (co2 === undefined || timestamp === undefined) {
    console.error("Invalid response format");
    return;
}
```

### Timestamp Validation

The `formatTime` function includes validation:

```javascript
const date = new Date(isoDate);
if (isNaN(date)) {
    console.error("Fecha no válida:", timestamp);

}
```

## Performance Considerations

### Memory Management

The application limits stored data to the last 10 readings:

- **Memory footprint**: ~200 bytes per reading (10 readings = ~2KB)
- **Chart performance**: 10 data points render instantly
- **Historical data**: Not persisted; lost on page refresh

### Polling Efficiency

- **Interval**: 5 seconds
- **Requests per minute**: 12
- **Requests per hour**: 720
- **Data transfer**: ~100 bytes per response
